## 功能

此技能用于帮助用户创建和维护项目的配置文档，主要功能包括：

1. 创建符合特定格式的 AGENTS.md 文档（中文）
2. 自动创建 CLAUDE.md 作为 AGENTS.md 的软链接
3. 提供默认的 重要说明 内容
4. 支持文档内容的更新和维护

## 使用场景

- 新项目初始化时创建配置文档
- 项目架构变更时更新文档
- 标准化开发流程时统一文档格式
- 需要为 Claude Code 提供项目配置信息时
- 用户提到"创建 AGENTS.md"、"更新项目文档"、"配置 Claude"等

## 文档格式

AGENTS.md 必须严格遵循以下格式和顺序：

```markdown
### Project: [项目名称]

[项目简介，包含技术栈和主要功能，一句话清晰描述]

---

### 规范

- [规范规则1]
- [规范规则2]
- [规范规则3]

---

### 常用命令

- `命令1`: [命令描述，包括端口等细节]
- `命令2`: [命令描述]
- `命令3`: [命令描述]

---

### 项目架构
```

[项目目录结构，使用树形格式]

```

---

### 重要说明

[重要说明和注意事项，包含默认内容]
```

## 默认内容

在 重要说明 部分必须自动添加以下默认内容：

- TDD驱动模式，完善的单测、集成测试
- 所有测试都需要使用真实环境，不要mock。例如，数据库使用真实数据库等
- 遇到问题优先使用 /systematic-debugging 彻底查明原因，再去解决。解决完之后，一定要验证、跑完所有测试才可以生成完成
- 端口号勿随意修改

## 完整执行流程

### 步骤 1：检查当前目录环境

首先查看当前目录的状态：

```bash
ls -la
```

### 步骤 2：收集项目信息

向用户询问以下信息（如果用户没有提供）：

1. **Project**: 项目名称和简短简介
2. **规范**: 规范规则（至少3条）
3. **常用命令**: 常用命令列表（至少3个）
4. **项目架构**: 项目架构描述或目录结构
5. **重要说明**: 项目特定的重要说明（可选，会追加到默认内容之后）

### 步骤 3：编写 AGENTS.md

根据收集的信息，严格按照格式编写 AGENTS.md。

#### Project 部分示例

```markdown
### Project: ShopFront

Next.js 14 电商应用，使用 App Router、支付功能和 Prisma ORM。
```

#### 规范部分示例（前端项目）

```markdown
### 编码规范

- TypeScript 严格模式，不使用 any 类型
- 使用命名导出而不是默认导出
- CSS 使用 Tailwind 工具类，不使用自定义 CSS 文件
- 使用 pnpm 管理依赖
- 代码格式化使用 Prettier
```

#### 规范部分示例（Python 项目）

```markdown
### 编码规范

- 遵循 PEP 8 规范
- 使用 uv 管理依赖
- 类型注解使用 mypy 检查
- 代码格式化使用 Black
- 导入顺序使用 isort
```

#### 常用命令部分示例

```markdown
### 常用命令

- `npm run dev`: 启动开发服务器（端口 3000）
- `npm run test`: 运行 Jest 测试
- `npm run test:e2e`: 运行 Playwright 端到端测试
- `npm run lint`: 进行 ESLint 检查
- `npm run db:migrate`: 运行 Prisma 迁移
```

#### 项目架构部分示例

```markdown
### 项目架构
```

├── src/
│ ├── app/ # Next.js App Router 页面路由
│ ├── components/ # 可复用组件
│ │ ├── ui/ # shadcn/ui 组件库
│ │ ├── navbar.tsx # 导航栏组件
│ │ └── theme-provider.tsx # 主题提供商
│ ├── hooks/ # 自定义 Hooks
│ ├── lib/ # 工具函数和工具库
│ └── types/ # TypeScript 类型定义
├── tests/ # 测试文件目录
│ ├── integration/ # 集成测试
│ └── unit/ # 单元测试
├── public/ # 静态资源
└── 配置文件

```

```

#### 重要说明部分

必须包含默认内容，然后可以添加用户提供的额外内容。

### 步骤 4：创建软链接

创建 CLAUDE.md 软链接指向 AGENTS.md：

```bash
ln -sf AGENTS.md CLAUDE.md
```

### 步骤 5：验证结果

检查文件是否创建成功：

```bash
ls -la | grep -E "(AGENTS\.md|CLAUDE\.md)"
```

验证软链接是否正确：

```bash
ls -la CLAUDE.md
```

## 更新现有文档的流程

如果用户想更新现有的 AGENTS.md：

1. 先读取现有的 AGENTS.md 文件
2. 询问用户需要更新哪些部分
3. 保留其他部分不变，只更新指定部分
4. 更新后重新验证软链接（如果需要）

## 注意事项

- **必须使用中文**编写整个文档
- **必须严格遵循格式**和顺序
- **不要省略**任何部分，即使内容较少
- 重要说明 部分必须包含默认内容
- 确保软链接创建成功
- 如果目录中已有 AGENTS.md，先询问用户是更新还是覆盖
- 文档内容要简洁准确，避免冗余
